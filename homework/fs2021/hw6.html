Background

Time flies, you all have been studying in the Codémon Academy for some time now. You have laughed so hard and also cried so much. The once strangers have now become your best friends in supporting your daily challenge of the learning and conquering of different amazing Codémons (though there are some that you are not really fond of...) Long story short, now is the time to have some fun away from the academy. And what’s better than engaging in a game to combat against your best friends and peers?!

Progarm Spcifications

In your program, you will create different “classes” in order to simulate the game that’s about to be described (either in this assignment or the next one).

Contestant Class

  • Member Variables
    • Each contestant should have a name
    • Each contestant should keep track of the number of Codémons owned
    • Each contestant should carefully maintain their pocket full of Codémons
      • the maximum number of Codémons they can carry is 10
    • Each contestant should also be carrying a backpack full of useful items
      • the maximum of items they can carry is 20
  • Member Functions
    • It should have a default constructor that will automatically prompt for the contestant’s name and set the number of Codémons owned to 0
    • It should have another constructor that accepts an argument for the name and set the number of Codémons owned to 0
    • It should also have a copy constructor. Why? Well so that perhaps the academy can secretly start cloning their students!
    • A function to update their pocket by adding a new Codémon info
    • A function to update their backpack by adding a new Item info
    • A prepare() function to get the contestant ready for the combat
      • It is to do two things: update the Codémons in the pocket and the items in the backpack
        • It should prompt the user for Codémon update one by one until the user doesn’t want to anymore or when the pocket is full; do the same for item updates
        • Note: when updating a Codémon, this function should also do the same for updating the skills list of each Codémon as well from the user
      • A print() function to output all of the relevant info of the contestant nicely: 
        Contestant name: Andy Myers 
Number of  Codémons owned: 2 
Pocket list: 
  1) Appletun 
  2) Blissey 
Backpack list: 
  1) Belue Berry 
  2) Cell Battery

The Codemon Class

  • Member Variables
    • Each Codémon should have a unique name (gifted by you)
    • Each Codémon should have the type info that it belongs to
    • Each Codémon should have a corresponding level (the higher the level the stronger it is)
      • Note: the level of a Codémon can only be a multiple of 5 (e.g. 0, 5, 10, 15, … )
    • Each Codémon should have a HP (health points) reading
    • Each Codémon should have a list of skills that it learned and acquired
      • the maximum number of talents each Codémon can have is 6
  • Member Functions
    • It should have a parameterized constructor that accepts arguments for the name, type info, level, and HP reading.
    • A function to update their skills list by adding a new Skill info
    • An evolve() function that cannot be triggered by any influence from the outside world; the Codémon has to be enlightened within
      • Upon the time of invocation (which might not happen during this assignment), it increases the level of the Codémon by 5

The Item Class

  • Member Variables
    • Each item should have a name
    • Each item should have a description of its purpose
    • Each item should also include the time duration of its effect once used. Be careful, some items could have a permanent effect!
  • Member Functions
    • It should have a parameterized constructor that accepts arguments for the name, purpose, and duration of the effect.

Skills Class

  • Member Variables
    • Each skill should have a name
    • Each skill should keep track of the amount of damage (base damage) that it could cause once executed
  • Member Functions
    • It should have a parameterized constructor that accepts arguments for the name and the base damage

Friend Functions

  • In this assignment, a global function called skillLookUp() is also needed to allow the user to search for a particular Codémon’s skill. It should also be considered as a friend function as well, but to which class(es)?
    • It should take just one parameter: the collection of all the contestants signed up for the combat
    • Within this function, it then should prompt the user for an input: the name of the skill to be looked up
    • Note: you can safely assume that there could be multiple Codémons possessing the same skill
    • If the search is successful, i.e. the skill exists among the contestants’ Codémons, then output the following message: 
      • Skill name: Bolt Strike
		Base damage: 130
		Acquired by: 
			1) Magneton (Andy Myers) 
			2) Togedemaru (Andy Myers)
